Version is an important concept in HD map services, which represents the evolution of data. To ensure the data in a catalog of certain version can represent a consistent view of the world and the convenience of data query and download, the following methods are used to manage versions.

• Catalogs, layers and partitions are versioned separately.


• The version of a partition represents the freshness of map data within a certain geographical range. When a specific partition is updated, its version is incremented by one.


• The version of the layer reflects the number of times it has been updated in batches (a batch can include a single or multiple partitions in the layer). When multiple partitions of a layer have been updated in a batch, the layer version is incremented by one.


• The version of the catalog reflects the number of times it has been updated in batches (a batch can include multiple partitions of any number of layers). When multiple partitions of multiple layers have been updated in a batch, Only the updated layer version is incremented by one, and the version of those layers which are not updated will remain the same. At the same time, although many layers have been updated, the catalog version is incremented only by one.


• The data version will not be updated after the data is uploaded successfully, and the version will not be updated until the publish is completed.


• Once a version has been published, the data in that version cannot be changed and removed. Data within a version is immutable and consistent.
