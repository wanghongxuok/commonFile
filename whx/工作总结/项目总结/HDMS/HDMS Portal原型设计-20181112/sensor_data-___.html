Application scenarios

As an important data source of HD Map updates and dynamic information, sensor data collected from the vehicles is essential for the freshness maintenance of map data and the safety of autonomous driving.

Scene 1： Live update HD Map freshness

• Vehicle: reality detection

• NavInfo: map learning from real-time sensor data

Scene 2：Dynamic Incident

• Vehicle: detection of incident ahead

• NavInfo: V to Center to X, Send information to improve the safety

Vehicle location

Absolute vehicle position information of several different accuracy, such as raw GPS position of lowest accuracy, lane-level matching to HD Map position

Status information

Include vehicle status, exceptional vehicle state, environment status, etc., mainly for obtaining dynamic event information

Map elements information

Include the information of linear elements ( such as lane boundary, road boundary) and nonlinear elements (such as sign, pole) in reality

Moving objects

Mainly include obstacle on the road, such as pedestrians and other vehicles. Same function with status information and can be used to create dynamic incident services

Sensor data can be used for a variety of purposes such as verifying and enhancing map attributes and also creating dynamic services. Such sensor data is typically originating in road vehicles and then submitted to HDMS Cloud through the OEM Backend or 3rd Party.

According to different elements, such as position estimate, vehicle status, environment status, traffic sign, etc., the requirements for sensor data are different.
