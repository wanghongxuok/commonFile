Get Map Data  

Data is uniquely identified through the datahandle. To get map data, you should get the datahandle first and then download the specific data from map service provided by NavInfo. We provide the Query API (for metadata) and Download API (for data) for you to get the map data you need.The Query API allows users to get metadata about layers and partitions stored in a catalog. The Download API allows users to download data to the specific path.

Publish Map Data  

You can use the PublishLayerPartitions API to publish data. Before publishing, it is essential for you to check the integrity of all the uploaded data and ensure that all the relevant data has been uploaded successfully. Once publishing, a new version number of layer and catalog will be generated and new data can be queried and downloaded by other users.

Upload Sensor Data  

 To upload sensor data, you need to obtain the information of what catalog and layer will be used to upload sensor data, and then transfer the crowdsourcing data collected from the vehicles to the Backend after campaign jobs has been done successfully, so as to accelerate the update of map data.You can use the Query API and UploadSensorData API provided by NavInfo to achieve your goals.

You can use the Upload API to upload map data to the specific layer of the specific catalog and check the correctness of the uploaded data. We provide six interfaces to ensure the correct uploading of data.

• Check whether the specified layer exists under the specified catalog, using the CheckLayerExist API.

• If you need to create a new layer to upload new data, using the CreateLayer API.

• If you need to check whether the uploaded data already exists in the layer, using the CheckDataExist API.

• Upload data to the specified layer under the specified catalog, using the UploadData API.

• Download the data you have already uploaded to review whether it is correct, using the DownloadData API.

• Delete the incorrect data which have already been uploaded, using the DeleteData API.
