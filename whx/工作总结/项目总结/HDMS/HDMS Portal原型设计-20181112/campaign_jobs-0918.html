Campaign Jobs

Overview

Content

Application scenarios

Campaign Jobs is for NavInfo partners through which requests for sensor data is being transported from HDMS Cloud to the OEM Backend or 3rd Party. As the demands for sensor data may be varying by vehicle configuration, location, time, date or even different scenarios, NavInfo generates different kinds of jobs to request the needed sensor data.

Common job

Full data collection job with less restrictions on category, area or time. It is based on a pre-agreed cooperation model. Primary purpose is for checking and improving quality of larger regions.

Validation job

Used to validate map element for Quality Control checks. The vehicle will report if a map element is present or not present.  If not present, NavInfo will trigger a separate Data Collection job.

Data collection job

Focused data collection job for a specific Map Element, Category or Area. This is typically triggered by a failed validation job or by production team based on their requirements.  This does NOT send back the raw sensor content and will only send CanBUS/"processed" content.

Raw Sensor data Collection job

This job will send to the backend the raw/actual sensor content (e.g. pictures, video, lidar, etc.)

Event trigger job

Full or partial data collection job based on dynamic incident report.  The content will include position, type of traffic incident and other map content.

HDMS distributes Campaign Jobs to OEM or 3rd Party, then to vehicle fleet.
